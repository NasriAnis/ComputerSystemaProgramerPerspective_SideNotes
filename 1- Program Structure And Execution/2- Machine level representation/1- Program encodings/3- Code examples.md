- To view the assembly code generated by the compiler we use the `-S` flag
- Using the `-c` flag the compiler will both compile and assemble the code

---

This code compiled using the `-S` flag :
```c
long mult2(long, long);
void multstore(long x, long y, long *dest) {
	long t = mult2(x, y);
	*dest = t;
}
```

translates to this :
![](../../../Attachements/Pasted%20image%2020260713111322.png)
but we ignore the line beginning with `.` since they are directives to guide the assembler and linker.

```asm
multstore:
	pushq %rbx
	movq %rdx, %rbx
	call mult2
	movq %rax, (%rbx)
	popq %rbx
	ret
```

![](../../../Attachements/Pasted%20image%2020260713111045.png)

We can also get the object file using the `-c` flag which gives a series of bytes that the computer translate to executions.

---

After getting object files `.o` the linker generate form those an actual executable file on which must contain a main function (starting point).

when compiling we can see that the executable is larger and that at the end there is `nop` instructions of for example the `multstore` function this is because they have been inserted to grow the code for the function to 16 bytes, enabling a better placement of the next block of code in terms of memory system performance.
